# Listing Localizer — 设计规格书

## 概述

面向 Shopee/Lazada 东南亚本地卖家的多语言 Listing 生成工具。卖家通过文字/链接/截图输入商品信息，一键生成适配 5 个市场的本地化标题、描述和关键词。

## 目标用户

印尼、泰国、越南、马来西亚、菲律宾的本地电商卖家。日常需要在 Shopee/Lazada 的多国站点上架同一商品，目前靠机翻或人工翻译。

## 产品形态

先做 Web 应用（FastAPI + HTMX + Tailwind），验证付费需求后追加 Chrome 插件。

## 核心功能

### 三种输入方式

| 方式 | 场景 | 实现 |
|------|------|------|
| 粘贴文字 | 卖家已有文案 | LLM 直接处理 |
| 商品链接 | 参考竞品/改编 | Playwright 抓取页面，提取标题、描述、图片描述 |
| 截图上传 | 手机端快速输入 | Vision LLM 识别商品属性 |

### 本地化生成

LLM 根据目标市场特性生成本地化版本，非字面翻译：

- **印尼** — 价格敏感话术、包邮关键词、清真相关标注
- **泰国** — 口语化表达、避免皇室相关用语
- **越南** — 年轻化用语、亲切感
- **马来西亚** — 英文混马来文
- **菲律宾** — Taglish 风格

每个市场输出：SEO 标题 + 产品描述 + 搜索关键词。

### 输出操作

- 页面直接展示结果
- 一键复制单个市场
- 一键复制全部（表格格式）
- 导出 CSV

### 用户系统（MVP 后加）

- 邮箱注册/登录
- 用量追踪
- 付费订阅

## 非功能需求

### 性能

- 文字输入：5 秒内返回结果（LLM 调用）
- 链接输入：15 秒内返回（含爬虫时间）
- 截图输入：10 秒内返回

### 安全

- URL 输入做白名单校验（仅允许 shopee.* / lazada.* 域名）
- LLM API Key 存服务端环境变量，不暴露前端
- 截图上传限制文件大小（5MB）

## 技术栈

- **后端**：Python 3.12+ / FastAPI
- **前端**：HTMX + Tailwind CSS
- **LLM**：DeepSeek API（文字处理）+ OpenAI/DeepSeek Vision（截图识别）
- **爬虫**：Playwright
- **部署**：单机即可，后续考虑 Railway/Render
